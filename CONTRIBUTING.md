# Contributing

Thank you for considering a contribution to Amulet Utility Plugins.

Contributions may include code, bug reports, feature requests, testing notes, documentation improvements, screenshots, examples, assets, compatibility information, or other project feedback.

## Before Contributing

Before opening an issue, discussion, or pull request, please:

1. Search existing issues and discussions to check whether the same topic was already reported.
2. Use the correct issue template or discussion category when possible.
3. Include relevant version information when reporting bugs.
4. Keep reports clear, practical, and focused.
5. Avoid sharing private worlds, private data, or files you do not have permission to share.

## Code Contributions

When contributing code, please try to:

1. Keep changes focused.
2. Avoid unrelated rewrites.
3. Explain what changed and why.
4. Test the plugin when possible before submitting.
5. Mention the Amulet version and Minecraft version used for testing.
6. Avoid adding unnecessary dependencies.
7. Keep compatibility with Minecraft Bedrock Edition and Amulet in mind.
8. Follow the existing code style where practical.

## Project Direction

The project maintainer may accept, reject, edit, reorganize, delay, close, or remove contributions when needed for compatibility, maintainability, safety, project direction, license concerns, or code quality.

Not every request or contribution will be accepted.

A contribution being discussed, reviewed, or tested does not guarantee that it will be merged.
